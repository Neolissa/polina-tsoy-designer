Coin · Внутренняя аналитика

Аналитический дашборд приложения

Понять, пользуются ли сотрудники платформой по-настоящему

Контекст

Relaunch корпоративной платформы для 100% сотрудников компании. HR и продукту нужны данные о вовлечённости — не только «сколько установили приложение».

Задача

Дашборд для внутренних стейкхолдеров: частота посещений, глубина просмотра, среднее время в приложении за сутки, разделы, пики по дням недели, опросы, оценки и удовлетворённость.

Роль

Product Designer: дерево метрик → дрилдаун периодов → когорты подразделений → связка поведенческих графиков с блоком опросов.

Подход

  • Дерево: поведение → контент → удовлетворённость
  • Дрилдаун: месяц → неделя → день; heatmap по дням недели
  • Глубина просмотра разделов: лента, обучение, геймификация, опросы
  • CSAT, NPS и таблица последних опросов рядом с активностью

Результат

Решения по развитию функций — социальные механики, обучение, геймификация — на данных, а не на ощущениях команды.

WUW · Аналитика

Дашборд аналитики

От разрозненных отчётов к одному экрану для продукта и партнёров

Контекст

B2C/B2B-приложение с геймификацией и white label. Команде нужны были воронки, удержание и сегменты — без бесконечных выгрузок в Excel и «сводок на коленке» перед каждым созвоном.

Задача

Спроектировать дашборд для PM, маркетинга и B2B-партнёров: за минуту видеть здоровье продукта, кастомизацию под бренд и узкие места в сценариях.

Роль

Product Designer: карта метрик с бизнесом → иерархия экранов → UI виджетов → согласование с разработкой на единой библиотеке блоков.

Подход

  • Карта метрик и приоритизация с продуктом и маркетингом
  • Иерархия: сводка → тренды → детализация по сценариям
  • Фильтры: период, сегмент, партнёр / white label
  • Переиспользуемые виджеты для dev — один паттерн KPI, график, таблица

Результат

Один источник правды для продуктовых решений; меньше времени на «собрать отчёт»; основа для A/B-тестов и развития white label.

DocsBird · ИИ в UI

Внедрение ИИ в интерфейс продукта

ИИ как помощник в документообороте, а не как «магическая кнопка»

Контекст

Международный документооборот: пользователям нужны подсказки по шагам, локализации и compliance — без перегруза интерфейса и потери доверия к продукту.

Задача

Встроить ИИ-сценарии в существующие потоки — создание соглашения, фильтры, подписание — предсказуемо, с контролем пользователя и возможностью откатить подсказку.

Роль

Product Designer: сценарии «человек в центре» → точки входа ИИ → состояния UI (ожидание, результат, ошибка) → мобильная адаптация.

Подход

  • Точки входа: подсказка, автозаполнение, проверка полей — в контексте формы
  • Прозрачность: чипы «предложено ИИ», кнопки «принять / отклонить»
  • Состояния: idle, loading, успех, ошибка — без блокировки всего экрана
  • Боковая панель ассистента с готовыми действиями, а не свободным чатом «на весь экран»

Результат

Меньше ошибок в формах, быстрее онбординг на новых рынках; ИИ усиливает сценарий, а не подменяет ответственность пользователя.
