Solution

Evolutionary refactoring instead of redrawing from scratch

1 I didn’t start redrawing everything from scratch

Chose an evolutionary refactoring strategy: gradually improved existing components, without breaking the working product. This made it possible to implement changes without downtime in development.

2 Selected the core components

From 200+ disparate elements I created a system from 35 reusable components. Each component is universal, expandable, with clear rules of application.

3 Introduced tokens instead of hardcode

Color, typography, and spacing became variable. One token change updates all screens. This opened up opportunities for white labeling and rapid experimentation with themes.

4 Designed gamification as a system

Holistic mechanics: Action → Points → Achievements → Reward. Each element has its own role in user retention.

What would you do differently?

Reflection and conclusions for future projects

⚡

Would have implemented Figma Variables sooner

At the beginning of the project they were in beta. This would make it easier to manage the design system from day one.

🧪

Would conduct A/B testing of gamification

Some mechanics turned out to be too difficult for users. Validation at an early stage would save time.

📚

I would create documentation for content managers

The developers were happy, but the content managers wasted time finding the right styles.
